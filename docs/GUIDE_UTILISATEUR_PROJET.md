﻿﻿﻿﻿﻿﻿﻿﻿﻿﻿﻿﻿﻿﻿﻿﻿﻿﻿﻿# GUIDE UTILISATEUR — MODULE “PROJECTS” (COMMUNITY INCUBATOR)

## 1) À quoi sert le module “Projects” ?

Le module **Projects** permet de transformer les discussions du **Forum GRIN17** en **projets communautaires structurés**, puis de mobiliser la communauté pour :
- commenter et améliorer le projet,
- voter (soutenir / s’opposer),
- faire remonter les projets les plus pertinents (tendance, populaires),
- aboutir à une **recommandation** et une **promotion** (Projet Ambassadeur).

Le but est de passer de l’idée à l’action avec un minimum d’intervention administrative.

## 2) Pré‑requis

- Être inscrit sur GRIN17.
- Être connecté pour :
  - soumettre un projet,
  - voter,
  - commenter,
  - convertir un post du forum en projet,
  - accéder au back‑office (administrateurs uniquement).

## 3) Vocabulaire & statuts (workflow)

Chaque projet a un **statut** qui décrit sa position dans le cycle de vie :

- **Brouillon (DRAFT)** : projet non publié, visible pour son auteur (selon les règles de la plateforme).
- **En discussion (IN_DISCUSSION)** : projet publié, ouvert aux commentaires et votes.
- **Validation communautaire (COMMUNITY_VALIDATION)** : le projet a atteint un seuil de votes et entre en phase de validation.
- **Recommandé (RECOMMENDED)** : la communauté a validé le projet (taux d’approbation élevé).
- **Projet Ambassadeur (AMBASSADOR_PROJECT)** : projet mis en avant par la plateforme (ex : sélection hebdomadaire).
- **Archivé (ARCHIVED)** : projet clôturé / retiré.

## 4) Accéder aux projets

### 4.1 Page “Projets”

Allez sur : **/projects**

Vous y trouverez :
- une **liste** des projets (cartes),
- des **modes** :
  - **Tous**
  - **Tendances**
  - **Populaires**
- des **filtres** :
  - Région
  - Statut
  - Tri (récents / popularité / tendance — selon le mode)

### 4.2 Comprendre une carte projet

Chaque carte affiche :
- le **titre**
- la **région**
- un **badge de statut**
- le nombre de **votes**
- le nombre de **commentaires**
- une barre de **taux d’approbation** (exprimée en %)

Cliquez sur une carte pour ouvrir le **détail du projet**.

## 5) Consulter un projet (page détail)

Sur la page **/projects/{id}**, vous trouverez :

### 5.1 Sections principales
- **Problème** : le besoin identifié dans la communauté
- **Objectifs** : les résultats attendus
- **Médias** :
  - images
  - documents
  - lien vidéo (si fourni)
- **Commentaires** : fil de discussion dédié

### 5.2 Colonne “Validation communautaire”

Vous verrez :
- le nombre total de votes
- le détail : soutiens / oppositions
- le **taux d’approbation**
- les boutons :
  - **Soutenir**
  - **S’opposer**

## 6) Voter sur un projet

### 6.1 Types de vote

Vous pouvez voter :
- **Soutenir** (support)
- **S’opposer** (oppose)

### 6.2 Effets

Votre vote :
- met à jour le compteur de votes,
- influence le taux d’approbation,
- participe aux calculs de **tendance** et de **promotion**.

### 6.3 Changer d’avis

Vous pouvez :
- changer votre vote (support ↔ oppose),
- ou annuler votre vote (en revotant la même option, selon la logique de la plateforme).

## 7) Commenter un projet

Dans la section **Commentaires** :
- saisissez votre message,
- cliquez sur **Publier**.

Bonnes pratiques :
- proposez des partenaires potentiels,
- signalez des risques,
- suggérez des améliorations mesurables (impact, indicateurs, calendrier).

## 8) Créer un projet (soumission via wizard)

Allez sur : **/projects/create**

La soumission se fait via un **wizard multi‑étapes** :

### 8.1 Étape 1 — Infos
Champs principaux :
- Titre
- Région
- Budget estimatif (FCFA)
- Description courte

### 8.2 Étape 2 — Impact & détails
Champs principaux :
- Problème à résoudre
- Objectifs
- Partenaires recherchés

### 8.3 Étape 3 — Médias
Vous pouvez ajouter :
- **Images** (illustrations, photos du contexte)
- **Documents** (notes conceptuelles, plans, budgets)
- **Lien vidéo** (YouTube/Vimeo) si pertinent

### 8.4 Étape 4 — Validation (récapitulatif)
Vous vérifiez les informations, puis vous choisissez :
- **Enregistrer brouillon** (DRAFT)
- **Publier** (IN_DISCUSSION)

Conseil : publiez lorsque le problème et les objectifs sont clairs — la communauté pourra ensuite améliorer le reste.

## 9) Convertir une discussion du forum en projet

### 9.1 Quand utiliser la conversion ?

Utilisez la conversion lorsque :
- un post du forum décrit un problème clair,
- la discussion génère des réactions/commentaires,
- l’idée mérite une structuration en projet.

### 9.2 Comment faire ?

1. Ouvrez le post forum : **/forum/post/{postId}**
2. Cliquez sur **Convertir en projet**
3. Vous êtes redirigé vers la page du projet créé

Le projet est pré‑rempli avec :
- le titre du post (ou un extrait)
- le contenu comme **problème**
- des objectifs initialisés (à affiner ensuite)

## 10) Suggestions “Idées du forum à fort potentiel”

Sur la page **/projects**, une section peut afficher des **idées à fort potentiel** issues du forum.

Pour chaque suggestion, vous pouvez :
- cliquer sur **Convertir** (redirige vers le post pour conversion),
- ou **Voir le projet** si déjà converti.

## 11) Gamification (score & reconnaissance)

Le module Projects contribue au score utilisateur :
- **Projet soumis** : +10 (lorsqu’il est publié)
- **Projet recommandé** : +25
- **Votes reçus** : +1 par vote reçu

Effets :
- progression du score,
- impact sur le classement/leaderboard (selon la configuration plateforme),
- visibilité et reconnaissance communautaire.

## 12) Notifications

Vous recevez des notifications quand :
- quelqu’un vote sur votre projet,
- quelqu’un commente votre projet,
- le statut du projet change (ex : validation communautaire),
- le projet est recommandé,
- votre post forum est converti en projet.

Accès :
- via la page **/forum/notifications** (ou l’icône notifications si disponible).

## 13) Tendances vs Populaires : comprendre la différence

- **Tendances** : privilégie l’activité récente (votes + commentaires récents).
- **Populaires** : privilégie le volume et le différentiel de votes (soutiens – oppositions).

Ces listes sont optimisées via cache pour rester rapides même avec un grand nombre d’utilisateurs.

## 14) Règles de bonne conduite (recommandations)

Pour un projet de qualité :
- décrivez un problème réel et localisé,
- fixez des objectifs concrets et mesurables,
- proposez des partenaires réalistes (collectivités, ONG, PME, écoles),
- ajoutez des médias utiles (photos, documents de cadrage),
- respectez la charte communautaire (contenu non haineux, non violent, non diffamatoire).

## 15) Back‑office (Administrateurs)

Accès : **/admin/projects**

Fonctionnalités principales :
- liste des projets
- recherche
- filtres : région / statut / tri
- actions :
  - **Recommander** (statut RECOMMENDED)
  - **Promouvoir Ambassadeur** (statut AMBASSADOR_PROJECT)
  - **Archiver** (statut ARCHIVED)

Bonnes pratiques admin :
- privilégier la promotion des projets déjà validés par la communauté,
- archiver les projets obsolètes ou hors charte,
- utiliser la recommandation pour valoriser les projets robustes et mobilisateurs.

## 16) Dépannage (FAQ)

### 16.1 Je n’arrive pas à voter / commenter
- Vérifiez que vous êtes connecté.
- Vérifiez que votre session est active (sinon reconnectez‑vous).

### 16.2 Je ne vois pas le bouton “Soumettre un projet”
- Ce bouton n’apparaît que lorsque vous êtes connecté.
- Accédez directement à **/projects/create** si besoin.

### 16.3 La page admin est inaccessible
- Vous devez avoir le rôle **Admin** ou **SuperAdmin**.

### 16.4 Pourquoi mon projet ne passe pas “Recommandé” ?
Le passage en recommandé dépend :
- du nombre de votes total,
- du taux d’approbation,
- et des règles automatiques de la plateforme.
